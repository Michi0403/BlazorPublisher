# v1.0.1 — workspace and timeline placement correction

- Restored the publisher canvas as the main flexible application row.
- Moved the page timeline from the ribbon stack into an on-demand bottom workspace pane.
- The timeline is hidden by default and can be opened or closed from the Animations, View, and Media Tools ribbon tabs.
- Media insertion and Media Studio completion open the timeline automatically for trimming and timing.
- Added timeline pane commands to page, object, page-navigator, and timeline context menus.
- Removed the workspace `height: 100%` override that could push the page surface and rulers out of the visible grid area.
- The timeline close command is available from its toolbar and every timeline context menu.
