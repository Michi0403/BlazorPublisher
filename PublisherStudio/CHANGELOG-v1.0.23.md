# PublisherStudio v1.0.23 story fidelity and command-surface completion

- Preserved the live RichEdit DOCX package directly when **Apply to frame** is used. Reopening a story now receives the same Office Open XML data that the already-correct DOCX download produces, preventing direct text color, highlighting, font, paragraph, table, and other formatting from being lost during the apply/reopen cycle.
- Added persistent RichEdit document-page background storage to text frames. PublisherStudio reads the page color directly from `word/document.xml` in the stored DOCX package, retains it in the publication model, and backfills it for older publications when possible.
- Applied the stored story page background consistently to the publication canvas, print publication, standalone story HTML, story printing, animated HTML export, and video export. The HTML preview converter now accepts the DOCX-derived color as the authoritative background instead of depending on whether RichEdit includes that color in its HTML export.
- Kept the complete formatted story preview self-contained by resolving RichEdit CSS to inline styles before it is stored. Text colors, highlights, fonts, paragraph formatting, lists, tables, borders, and spacing therefore use one preview representation across canvas and publication exports.
- Expanded the publication right-click animation menu with the full entrance, emphasis, motion, exit, media-control, preview, timeline, and clear-animation command families. Page transitions and page-animation preview are also available from the page context menu.
- Expanded Picture Studio right-click editing across raster, text, paint, procedural-render, shape, fill, and common layer properties. Procedural renders expose renderer type, colors, scale, detail, softness, contrast, angle, stripe width, random seed, reset, and a direct jump to the full property editor.
- Made the active Picture Studio property target explicit with a selected-layer heading, layer glyph, layer description, and highlighted procedural-render property section.
- Added lightweight built-in icons to every DevExpress ribbon command in PublisherStudio, Story Editor, Picture Studio, Media Studio, Barcode Studio, data editors, and the timeline. Drawing, selection, render, animation, media, data, page, clipboard, and arrangement commands now remain visually distinguishable without adding an external icon package.
- Retained the v1.0.22 mouse selection, grouped cross-layer movement, desktop clipboard, keyboard clipboard, drag/drop, recovery, chart stability, and HTML/video presentation-export behavior.
- Kept `using Microsoft.AspNetCore.Components.Server;` in `Program.cs` for the existing `CircuitOptions` configuration.
- Corrected the v1.0.23 source package build errors: dynamic DevExpress context-menu labels now come from strongly typed string helpers, and the standalone Story HTML template uses a two-dollar raw interpolated string so CSS braces remain literal.
- Publication format marker updated to `1.23`; older publications remain loadable.
