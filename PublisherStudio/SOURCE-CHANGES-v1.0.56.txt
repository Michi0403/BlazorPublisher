PublisherStudio v1.0.56 source changes
======================================

Baseline
--------
Compared with the supplied/repaired PublisherStudio v1.0.55 source tree.
The source archive contains no .git directory, so commit-level Git history was
not available. Existing release/changelog history was preserved.

Functional repairs
------------------
1. Repeatable animation Preview
   - publisherInterop.js clears the prior preview and captures stable per-node
     base transforms before scheduling animations.
   - Inline authored transforms take precedence over computed Web Animations
     matrices, preventing scale/rotate/translate accumulation on repeated runs.
   - Group members receive their own captured baseline.
   - timelineInterop.js and the embedded standalone/export runtime use the same
     inline-first transform rule.

2. Publication-owned standard settings
   - Zoom, ruler unit, grid/guides, all snapping options, grid spacing, and
     export DPI continue to live in PublicationDocument and now mark the
     publication dirty when changed, so Save and recovery include them.
   - No global application preference store was introduced for these settings.

3. Local protected streaming settings
   - PublicationFileService removes only the streaming property from saved,
     recovery, and undo JSON.
   - PublicationStreamingSettingsStore persists per-publication streaming
     configuration under Local Application Data using ASP.NET Core Data
     Protection.
   - Existing publication files containing embedded streaming configuration are
     migrated once to the local store.
   - Undo/redo preserves the active local streaming configuration.

4. Professional Components Map drag stability
   - Designer-mode Map hosts receive a transparent gesture shield.
   - Canvas capture-phase selection, movement, double-click, and context-menu
     handling remain active, while the map provider cannot pan/zoom from the
     same gesture.
   - Presentation/export mode and VectorMap remain unchanged.

5. Version and regression coverage
   - Application, installer, npm package, and integrated streaming capability
     version aligned to 1.0.56.
   - Added publicationPersistence.test.mjs and included the existing pointer
     ownership suite in the main npm test chain.
