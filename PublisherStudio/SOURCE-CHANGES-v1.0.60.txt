PublisherStudio v1.0.60 source changes
======================================

Repair scope
------------
This repair addresses a Gallery that could advance several items from one
navigation-button press in the main publication designer and exposes both
application canvas zoom renderers in the mainframe ribbon. CSS layout zoom stays
the default; the previous transform renderer remains selectable.

Deterministic designer Gallery navigation
-----------------------------------------
src/PublisherStudio.Web/wwwroot/js/componentRuntime.js
src/PublisherStudio.Web/wwwroot/js/publisherInterop.js
- Disabled DevExtreme Gallery swipe recognition only in designer mode, where the
  publication canvas already owns drag gestures.
- Removed designer Gallery transition overlap by using immediate navigation.
- Preserved the current selected Gallery index across harmless runtime rerenders.
- Gave native DevExtreme controls inside a selected Professional Component full
  pointer ownership. Gallery buttons/indicators, buttons, tabs, menu items, form
  controls, sliders, and scrollbars no longer start a simultaneous canvas drag.
- Kept the first control click on an unselected component selection-only, which
  prevents an accidental action while preserving normal design-tool selection.
- Retained native Gallery swipe and animation in export/runtime mode.

Selectable publication canvas rendering
----------------------------------------

- Added persisted PublicationCanvasZoomMode values CssLayout and Transform.
- Added a View > Zoom ribbon selector with checked Sharp CSS layout (default) and
  Compact transform options.
- Included the mode in canvas reinitialization/layout keys and exposed it to the
  browser runtime through page/config data.
- Resynchronized all editor object content wrappers when zoom mode changes, so
  stale inline zoom/transform values cannot survive a mode switch or live resize.
- Kept ordinary publication content on Chromium CSS layout zoom when selected and
  supported, with transform scaling as the compact mode and browser fallback.
- Kept DevExtreme Professional Components on the transform-compatible editor path
  in both modes to avoid CSS-zoom coordinate conflicts in Gallery, Map, Vector
  Map, virtual scrolling, and other widget internals.
- Left PrintPublication and all export paths unchanged.
- Advanced publication format from 1.46 to 1.47. Invalid/missing mode values
  normalize to the CSS default.
