PublisherStudio v1.0.62 source changes
========================================

Scope
-----
This release applies the agreed repository architecture before future Studio interchange work and rebuilds the integrated streaming backend inside the existing PublisherStudio.Web monolith.

No Picture Studio, Video Studio or Audio Studio interchange adapter is activated in the UI in this release. The format capability matrix and adapter/loss-report contract are now version-controlled so the first format can be implemented as a deliberate Studio-owned vertical slice rather than a Mainframe-wide importer.

The contract defines Components, Controllers, Services, HostedServices, Backend, Domain and Models as the stable roots. UseCases is allowed only beneath an owning root for orchestration. Main-host application routes use MVC controllers. Backend code may not depend on MVC/controllers. Provider credentials and machine settings remain outside publication/interchange files.

Streaming implementation ownership
----------------------------------
Moved low-level implementations into Backend/Streaming:
- Capture: native registry, device discovery and Windows process loopback
- Chat: provider Chat hub and Twitch/YouTube adapters
- Encoding: encoder orchestration and FFmpeg discovery
- LAN: LAN server, RTSP and WebRTC signaling
- Metadata: Now Playing/ID3 reader
- Sessions: runtime MediaSession state

Moved long-running work into HostedServices/Streaming:
- GlobalHotkeyService
- TwitchOAuthMaintenanceService

Moved application services into structured Services/Streaming subnamespaces:
- MediaHost/StreamingMediaHostClient.cs
- OAuth/TwitchOAuthService.cs
- Profiles/StreamingProfileStore.cs
- Settings/PublicationStreamingSettingsStore.cs
- Sessions/MediaSessionRegistry.cs
- StreamingServiceCollectionExtensions.cs

Shared streaming request/event contracts now live in Domain/Streaming/StreamingRuntimeContracts.cs.

StreamingMediaHostClient now delegates device discovery and session lifecycle through StreamingRuntimeUseCases and StreamingSessionUseCases instead of directly reaching into NativeDeviceDiscovery or MediaSessionRegistry.

Compatibility and security
--------------------------
- The integrated runtime remains in PublisherStudio.Web; no Media Host executable or microservice was added.
- Existing Twitch OAuth, provider Chat, native capture, FFmpeg, recording, separate stop controls, WebRTC/HLS/RTSP/LAN, hotkey and Now Playing behavior is retained.
- Protected provider profiles, OAuth sessions, stream keys, LAN secrets and recording paths remain local and outside publication/template/interchange JSON.
- Publication format remains 1.47 and Picture Studio format remains 1.2.
