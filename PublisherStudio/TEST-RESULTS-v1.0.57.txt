PublisherStudio v1.0.57 validation results
==========================================

Baseline and scope
------------------
Baseline: repaired PublisherStudio v1.0.56 source archive.
The archive contains no .git directory, so commit-level Git history was not
available. Existing changelog and release history was retained.

Reported regression
-------------------
At reduced editor zoom, picture geometry scaled but RichEdit text reflowed and
native audio controls retained their 100% layout. The editor was resizing every
object frame in CSS pixels without applying the same visual zoom to the complete
object subtree. Absolute font sizes, native controls, and responsive component
layouts therefore behaved as if the object itself had been resized.

Repair contract
---------------
- Outer publication object geometry remains zoom-aware for drag, resize,
  snapping, selection, connector ports, and connector routing.
- Every object's authored subtree is laid out at stable 96-DPI dimensions and
  receives one editor-only scale transform.
- Millimeter-based inner padding, borders, masks, and strokes use canonical
  96-DPI pixels before that transform to avoid double scaling.
- Live resize and cancelled-resize restoration synchronize the stable content
  wrapper before recalculating content fit.
- PrintPublication and export surfaces do not use the editor-only wrapper.
